# 常见问题

## 一般问题

??? question "我的数据是否存储在外部服务器上？"
    不会。所有内容都在您的服务器上本地运行。简历数据存储在 `/data` 目录中的 SQLite 数据库文件中。

??? question "可以不用 Docker 运行 CV Manager 吗？"
    可以。安装 Node.js 18+，在项目目录中运行 `npm install`，然后执行 `node src/server.js`。管理界面运行在端口 3000，公开站点运行在端口 3001。

??? question "多人可以使用同一个实例吗？"
    CV Manager 设计为单用户应用。每个实例管理一个人的简历。如需多人使用，请运行独立的容器。

## 编辑

??? question "如何将一个职位标记为'在职'？"
    将 **End Date** 字段留空即可。简历上将显示为"Present"。

??? question "可以对分区内的项目进行排序吗？"
    可以。大多数项目支持拖拽排序，排序结果会自动保存。

??? question "如何为工作经历添加要点？"
    编辑该工作经历，在 **Highlights** 字段中输入要点——每行一个。

??? question "如何添加公司 Logo？"
    编辑工作经历，滚动到 **Company Logo** 部分，点击 **Choose image** 上传。也可以点击 **Use existing** 复用已上传的 Logo。启用 **"Sync logo across all [Company]"** 开关可将同一 Logo 应用到该公司的所有工作经历。

??? question "我不小心删除了某些内容，能撤销吗？"
    目前没有撤销功能。由于编辑内容会自动保存到活动数据集，更改会立即持久化。如果您有之前的导出文件或其他已保存的数据集，可以从中恢复。建议定期导出简历作为备份。

## 自定义分区

??? question "可以创建多少个自定义分区？"
    没有硬性限制。根据需要创建即可。

??? question "创建后可以更改自定义分区的布局类型吗？"
    可以。编辑该分区并选择不同的布局。请注意，某些字段在不同布局类型之间可能无法保留（例如从卡片切换到社交链接）。

??? question "'Bullet Points'和'Free Text'布局有什么区别？"
    **Bullet Points** 将每一行渲染为带有组标题的项目符号列表项。**Free Text** 渲染纯文本，保留换行且没有标题——类似于"关于/简介"分区。

## 打印与 PDF

??? question "为什么我的 PDF 和屏幕显示不同？"
    打印输出使用专门为纸张优化的打印样式。某些视觉效果（悬停状态、动画、渐变）会被简化。隐藏项目和管理控件会自动移除。

??? question "如何让简历占用更少的页数？"
    尝试在 Print & Export 设置中启用 **Allow Section Splits** 和 **Allow Item Splits**。也可以隐藏不太重要的项目或分区，或使用更紧凑的自定义分区布局。另外还可以通过浏览器的打印对话框调整缩放比例（有时这个选项比较隐蔽）。

??? question "为什么打印的简历中缺少某些项目？"
    请检查这些项目是否已被切换为隐藏（眼睛图标）。隐藏项目会从打印输出和公开视图中排除。

??? question "页码没有显示？"
    请确保在 Settings → Print & Export 中启用了 **Page Numbers**。某些浏览器 PDF 查看器可能无法显示 CSS 生成的页码——请尝试下载 PDF 并在专用阅读器中打开。

## 时间线

??? question "时间线显示的日期不对/只显示年份/显示完整日期？"
    时间线有独立的日期设置。前往 **Settings → Advanced → Timeline: Years Only** 切换仅显示年份或完整日期格式。

??? question "可以直接在时间线中添加条目吗？"
    不可以。时间线是根据工作经历自动生成的。添加或编辑工作经历后，时间线会相应更新。

??? question "时间线上的国旗没有显示？"
    请确保工作经历的 **Country Code** 字段设置为有效的 2 位 ISO 国家代码（例如 `us`、`gb`、`ch`、`de`、`fr`）。国旗图标从外部 CDN 加载。

??? question "同时有两份工作时会怎样？"
    时间线会自动检测重叠的职位，并将其渲染为**并行轨道**。同期的工作会出现在一条升高的分支线上，以 S 曲线连接器显示分叉和合并点。无需任何配置——完全基于您的开始/结束日期。短于 1 个月的重叠会被忽略（这在职位过渡期间很常见）。

??? question "为什么时间线显示的是 Logo 而不是公司名称？"
    如果您为该工作经历上传了公司 Logo，时间线会显示 Logo 图片而非文字。如果 Logo 文件缺失，则会回退显示公司名称。要从时间线中移除 Logo，请编辑该工作经历并在 Company Logo 部分点击 **Remove**。

## 语言与更新

??? question "如何更改管理界面语言？"
    点击工具栏中的**地球图标**，从下拉网格中选择语言。更改会立即生效并跨会话保存。

??? question "如何查看当前运行的版本？"
    打开 **Settings**——版本号显示在模态框的左下角（例如 `v1.11.0`）。

??? question "新版本已发布但我没看到更新横幅？"
    版本检查有 24 小时缓存。重启服务器（或 Docker 容器）可清除缓存并强制重新检查。您的服务器还需要能够访问 `raw.githubusercontent.com` 的出站网络连接。

## 数据集 / 多份简历

??? question "什么是'Default'数据集？"
    默认数据集是访客在根 URL（`/`）看到的简历版本。首次安装时，CV Manager 会自动从您的简历数据创建一个"Default"数据集。您可以随时使用 Open 模态框中的单选按钮更改默认数据集。

??? question "我的编辑会自动保存吗？"
    是的。您在管理界面中的每项更改（添加、编辑、删除、排序、切换可见性）都会在短暂延迟后自动保存回活动数据集。横幅会显示"Saving..."然后显示"Saved"以确认。

??? question "'Load'数据集时会发生什么？"
    加载数据集会将您的工作副本切换到该数据集。之前的编辑已经自动保存，因此不会丢失任何内容。

??? question "访客能实时看到我的编辑吗？"
    不能。公开站点提供的是冻结的默认数据集，而非您的实时编辑。访客只有在自动保存将更改写入默认数据集后才能看到变化。如果您正在编辑非默认数据集，访客在您将其设为默认之前完全看不到这些更改。

??? question "访客能看到我保存的数据集吗？"
    只有设为公开的才能看到。每个数据集在 Open 模态框中都有一个切换开关。设为公开后，该版本可通过公开站点（端口 3001）的 `/v/slug` 访问。私有数据集只能从管理界面预览。

??? question "如何与他人分享特定的简历版本？"
    打开 **Open...** 模态框，将数据集切换为公开，然后点击 slug URL 旁的复制图标。分享该链接——它在公开站点上可用，无需暴露您的管理界面。

??? question "可以同时有多个公开版本吗？"
    可以。您可以将任意多个数据集设为公开。每个都有自己的 URL（例如 `/v/technical-cv-1`、`/v/marketing-cv-2`）。主页面 `/` 显示默认数据集。

??? question "可以删除默认数据集吗？"
    不可以。当前被选为默认的数据集（通过单选按钮）无法删除。请先将另一个数据集设为默认，然后再删除旧的。

??? question "搜索引擎会索引我的版本化 URL 吗？"
    默认不会——版本化页面带有 `noindex, nofollow`。要允许索引，请在 Settings → Advanced 中启用 **Index Versioned URLs**。

## 公开站点与 SEO

??? question "如何分享我的简历？"
    分享您的公开服务器（端口 3001）的 URL。如果您已通过 Cloudflare Tunnel 或反向代理设置了域名，请分享该域名。根 URL 始终显示您的默认数据集。您还可以使用公开版本化 URL 分享特定版本（参见[数据集](../guide/datasets.md)）。

??? question "搜索引擎会索引我的简历吗？"
    默认会。主公开页面包含适当的 meta 标签、站点地图和 robots.txt。要阻止索引，请在 Settings → Advanced 中将 **Search Engine Indexing** 设置为"No Index"。公开版本化 URL（`/v/slug`）默认**不被索引**；如需搜索引擎抓取，请启用 **Index Versioned URLs**。

??? question "可以为简历添加 Google Analytics 吗？"
    可以。在 **Settings → Advanced → Tracking Code** 中粘贴跟踪代码。它仅注入到公开页面中。

## Docker 与基础设施

??? question "我的更改没有出现在公开站点上？"
    公开站点提供的是**默认数据集**，在您于管理界面编辑时会自动更新。尝试在公开站点上进行硬刷新（`Ctrl+Shift+R`）。如果运行独立容器，请确保它们共享同一个数据卷。

??? question "出现'port already in use'错误？"
    请更改 Docker 配置中的宿主机端口映射。例如，映射为 `3010:3000` 和 `3011:3001`。**不要** 更改 `PUBLIC_PORT` 环境变量——那是容器内部端口。

??? question "如何备份数据？"
    两种方式：使用管理工具栏中的 **Export** 按钮（导出 JSON），或备份包含 SQLite 数据库和上传图片的 `data/` 目录。

??? question "头像没有显示？"
    请确保图片是通过管理界面上传的。文件存储在 `data/uploads/picture.jpeg`。如果在 Linux 上运行，请检查文件权限。
