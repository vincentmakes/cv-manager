# 数据集（多版本简历）

## 数据集的工作原理

数据集是您简历的保存快照。其中一个数据集始终是**默认**的 — 这是访客在您的根 URL（`/`）看到的版本。您可以为不同受众创建额外的数据集（例如技术简历、管理简历），并通过各自的 URL 分享。

首次安装 CV Manager 时，系统会自动从您的简历数据创建一个"默认"数据集。您在管理端所做的所有编辑都会**自动保存**回当前活动的数据集 — 无需单独的"保存"步骤。

## 活动数据集横幅

工具栏下方的横幅显示您当前正在编辑的数据集。它显示：

- **数据集名称**（例如"默认"、"技术简历"）
- 如果该数据集是 `/` 上展示的公开版本，则显示**"默认"徽章**
- **自动保存状态** — 每次编辑后短暂显示"保存中..."然后显示"已保存"

您所做的每项更改（添加条目、编辑内容、重新排序、切换可见性）在短暂延迟后会自动保存到活动数据集。

## 保存新数据集

点击工具栏中的**另存为...**，输入名称（例如"技术简历"、"市场岗位"），您当前的简历状态将保存为新快照。新数据集将成为活动数据集。

## 打开弹窗

点击**打开...**查看所有已保存的数据集。顶部的**图例**解释了三个控件：

| 控件 | 用途 |
|------|------|
| **单选按钮** | 选择哪个数据集作为根 URL `/` 上展示的默认数据集 |
| **开关** | 将其他数据集通过各自的 `/v/slug` URL 分享 |
| **眼睛按钮** | 预览已保存的数据集，而不将其公开 |

每行数据集显示：

- **名称**和最后更新日期
- **"默认"徽章** — 显示在通过单选按钮选中的数据集上
- **"编辑中"徽章** — 显示在当前加载到管理端的数据集上
- **版本化 URL**（例如 `/v/technical-cv-1`）— 默认数据集不显示此项，因为它直接展示在 `/` 上
- **加载**按钮 — 切换到此数据集（如果已是活动数据集则显示"重新加载"）
- **删除**按钮 — 永久删除该数据集（当前默认数据集不可删除）

## 设置默认数据集

默认数据集是访客访问您的根 URL（`/`）时看到的版本。要更改它：

1. 打开**打开...**弹窗
2. 点击您想设为公开简历的数据集旁边的**单选按钮**
3. 更改立即生效 — 公开站点将展示该数据集

这将您的公开简历与编辑工作解耦。您可以在管理端自由编辑内容，访客不会看到进行中的更改，直到您准备就绪。

## 公开版本化 URL

每个已保存的数据集（默认数据集除外）都有一个唯一的 URL 路径（例如 `/v/technical-cv-1`）。默认情况下，这些是**私有的** — 仅可从管理界面访问用于预览。

要公开分享特定版本：

1. 打开**打开...**弹窗
2. 找到您想分享的数据集
3. 切换旁边的**开关** — 它变为蓝色并出现绿色的**公开**徽章
4. `/v/slug` URL 现在可以在**公开站点**（端口 3001）上访问

这让您可以与不同受众分享定制的简历版本。例如，您可以将"技术简历"公开用于工程岗位，同时将"管理简历"保持私有直到需要时再公开。

**复制 URL**：点击 slug 旁边的复制图标将完整 URL 复制到剪贴板。提示消息会告诉您复制的是公开 URL 还是仅供预览的 URL。

!!! note "注意"
    `/` 上的主要公开页面始终显示**默认数据集** — 而不是您的实时编辑。这意味着您可以在管理端安全地进行实验，而不会影响访客看到的内容。
